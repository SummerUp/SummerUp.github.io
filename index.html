自动转单工具

项目需求

公司越来越多的产品逐渐开拓海外市场，我也加了一个海外运营的项目。因为合作方使用的是另外一套bug管理系统（JIRA），作为QA我需要同步两个系统的bug信息，不断重复的“复制–粘贴”操作让我疲于应付，完全没有精力开展其他更有意义的工作。因此，我决定写一个自动转单的工具，能够自动检测JIRA上的bug单，并转单到公司内部的bug管理系统（Redmine）。

实现框架

我将工具的实现主要分成3个层次，最底层的是logger.py和config，分别负责记录运行日志和方便用户配置。中层的4个模块（recv_mail.py、html_parser.py、buglist.py、robot.py）是工具的主体，实现了工具的基本功能，并为上层提供调用接口。上层的main.py模块用于实现实际的处理逻辑，即遇到各种情况要如何操作。这样的设计较好地剥离了各个模块的相关性，易于实现优化和调整，同时具有较好的扩展性。

接口说明

首先介绍底层的两个模块：

  1. logger.py利用logging库实现，能够分级别地记录运行日志，方便用户调试。

    使用方法如下：

  2. config用于填写用户的配置信息，主要包括[email]、[area]、[popo]、[redmine]、[record]几个部分，具体内容不详细介绍。

接下来着重介绍中间层的调用接口：

  1. recv_mail.py

    负责监测JIRA上项目bug的变动，包括新建bug和修改bug等，提供接口如下：

    • MyMailbox.get_password() # 引导用户输入账号&密码
    • MyMailbox.recv_mail() # 完成登录邮箱、检查未读邮件、读取邮件等操作
    • MyMailbox.parse(msg) # 负责解析邮件内容，msg是读取的邮件内容
    • MyMailbox.get_lastUid() # 获取最后一封处理过邮件的id

  2. html_parser.py

    负责从JIRA网站抓取bug内容，提供接口如下：

    • MyUrlParser.parse(bug_id) # 根据bug_id合成对应的url，并抓取相应bug信息
    • MyUrlParser.clear(bug_id) # 清楚MyUrlParser实例的成员数据

  3. bugList.py

负责存储bug信息，采用内存+DB的模式，既保证运行速度，也避免了出现异常后的信息丢失。提供的接口如下：

  • BugList.check_open(bug_id) # 查询bug_id是否在未关闭bug的列表中
  • BugList.check_close(bug_id) # 查询bug_id是否在未关闭bug的列表中
  • BugList.add(bug_id, rdm_id) # 新增bug
  • BugList.close(bug_id, rdm_id) # 关闭bug单
  • BugList.get(bug_id) # 获取bug信息
  • BugList.show() # 打印所有bug信息到终端
  • BugList.logout() # 打印所有bug信息到log
  • BugList.get_bug_string() # 将所有bug信息转成string格式返回，可用于popo通知
  • DBHandle.add(bug_id, rdm_id) # 增加bug到DB

  1. robot.py

    负责调用redmine接口和popo通知，提供的接口如下：

    • Robot.create_issue(subject, description=’RT.’, attachment=[]) # 提单到redmine的接口
    • Robot.updated_issue(issue_id, version = “”, status = “”, subject = “”, description = “”, assign = “”) # 更新redmine上的bug单
    • Robot.query_issue(key) # 查询redmine接口
    • Robot.send_popo(msg) # popo通知接口

什么是virtualenv

当我们需要在同一台设备上使用Python开发多个项目时，每个项目需要依赖的库都不相同，因此每个项目可能需要各自独立的Python运行环境。Virtualenv就是用来为各个项目创建独立的Python环境，使多个Python应用相互独立互不影响。

virtualenv有如下几个功能：

  • 在没有权限的情况下安装新套件
  • 使得不同应用可以使用不同的套件版本
  • 某应用的套件升级不影响其他应用
